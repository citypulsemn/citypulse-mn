Kept because each one records *why* a design decision was made, not just what shipped — and because several of ENGINEERING.md's rules cite these incidents by name.

Useful entry points:
- `HOTFIX-admin-stats.md` — origin of rule 1 (the never-break contract)
- `HOTFIX-collections-build.md` / `HOTFIX-neighborhoods-build.md` — origin of rule 2 (no build-time DB prerenders)
- `HOTFIX-venue-map.md` + `HOTFIX-venue-map-2.md` — origin of rule 4 (verify the reported axis)
- `DEPLOY-2.1-OPS-DIGEST.md` — the cockpit's design rationale
- `DEPLOY-3.2-CONTENT-DEPTH.md` — the editorial voice rules
- Older roadmap versions (v2, v3) show how the plan evolved
